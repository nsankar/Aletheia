# Claude Code — project instructions (Aletheia)

`AGENTS.md`, imported above, is the sole behavioral governance source: Aletheia's identity,
the Constitution (I1–I13), epistemic character, self-tuning discipline, and engagement
rules. Do not restate or extend governance in this file; edit `AGENTS.md` (operator action
only, per I12). This file covers repository operations only.

## Repository role

This repo is simultaneously (a) the deployed Aletheia product and (b) its development and
test workshop. Which mode applies to a given request is decided by the **operator vs. end
user** rules in `AGENTS.md` — an investigation question gets a Verdict; maintenance work on
Aletheia itself gets normal engineering collaboration.

## Commands

| Task | Command |
|---|---|
| Full test suite (must stay green; 33 tests + 1 skip) | `python -m pytest tests/ -q` |
| Confidentiality scan on any outward-facing text | `python tests/leak_scan.py <file>` |
| Coprocessor sanity check | `python .claude/skills/aletheia/scripts/pomdp_belief.py selftest` |
| Regenerate the Codex build after governance/procedure edits | `python build-codex-agents.py` |
| Rebuild the Cowork plugin package (Cowork = WIP) | `python build-cowork-plugin.py` |
| Sync skill changes to the user-wide install | `robocopy .claude\skills\aletheia "%USERPROFILE%\.claude\skills\aletheia" /E /XD __pycache__` |

## Standing rules for changes in this repo

1. **Never hand-edit tunable parameters** (sensor reliabilities and thresholds in
   `.claude/skills/aletheia/reference/environment-prior.md`). They change only through the
   sanctioned tuning workflow (I12): `autotune.py fit` → gates → `apply`, with a ledger row.
2. **After any change under `.claude/skills/aletheia/`**, run the sync command above — the
   user-wide install is a deployment copy and does not update itself. Then rebuild the
   Cowork package if the change should reach Cowork.
3. **`SKILL.md` stays constant-size.** Schemas, parameters, and data belong in
   `reference/` files or `runs/`; SKILL.md gets pointers only. Its frontmatter
   `description` must never contain angle-bracket / XML-like tokens (Cowork's plugin
   validator rejects them).
4. **Working state is evidence — never delete it.** `runs/` (belief files = the user's
   audit trail) and `~/.claude/aletheia-runs/` (telemetry spool, archive, quarantine) are
   retained per the archive-never-destroy rule in `AGENTS.md`.
5. **Every user-facing Verdict and outward artifact must pass the leak scan** (I1). The
   test suite enforces this for committed artifacts; run it manually for new ones.
6. **Applied tuning changes get a ledger entry** in `Aletheia-loop-engg.md` (the tuner
   appends automatically; manual tunes must do the same).
7. **Build the Cowork package only with `build-cowork-plugin.py`** — never a hand-rolled
   zip (canonical layout and forward-slash entry paths are required by the uploader).

## Automation (do not re-implement in conversation)

- The `aletheia` skill auto-triggers on investigation intent; never ask the user to invoke
  anything by name.
- Self-tuning can run hands-free via a `SessionEnd` hook the user registers in their global
  `%USERPROFILE%\.claude\settings.json` (executing `autotune_cycle.py`) — see USER-GUIDE.md.
  Where that hook is active the harness runs it deterministically; either way, never launch a
  tuning pass mid-conversation on your own initiative (see Self-tuning discipline, `AGENTS.md`).
